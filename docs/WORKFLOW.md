# WORKFLOW.md — 작업 유형별 흐름 예시

> 작업 유형별로 어떤 순서로 진행하는지 참고용 흐름을 정리한 문서.
> 세션 시작 필수 읽기 순서는 `AGENTS.md` 참조.

---

## 기능 구현 요청

```
1. docs/pages.md 확인 (해당 페이지가 있는지)
2. docs/conventions.md 확인 (컴포넌트/훅/스토어 규칙)
3. docs/design-system.md 확인 (UI 작성 규칙)
4. 설계 설명 → 승인 대기 → 코드 작성
5. docs/progress.md 업데이트
```

## 버그 수정 요청

```
1. 근본 원인 분석
2. 수정 근거 설명 → 승인 대기 → 코드 작성
3. 관련 docs 업데이트 (필요시)
```

## 문서 수정 요청

```
1. 실제 코드와 문서 내용 비교
2. 현재 상태 vs 목표 상태 구분
3. 수정 후 관련 문서 크로스체크
```

## PR 생성 요청

```
1. .github/PULL_REQUEST_TEMPLATE.md 읽기
2. docs/GIT_WORKFLOW.md 브랜치/커밋 규칙 확인
3. 변경 파일 목록 확인
4. 템플릿 형식 그대로 PR 작성
```
