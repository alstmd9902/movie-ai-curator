# WORKING_STYLE.md — AI 작업 방식 공통 지침

> 모든 AI 코딩 도구가 코드 작성 전 계획, 승인, 문서 동기화 기준을 동일하게 따르도록 정리한 문서.

> CLAUDE.md와 AGENTS.md가 이 파일을 공통 출처로 참조한다.

## 파일 추가 위치 규칙

이 프로젝트는 **Claude Code + Codex + Cursor**를 함께 사용한다.
파일을 추가하기 전에 반드시 아래 기준으로 위치를 판단한다.

| 판단 기준 | 위치 |
|-----------|------|
| 모든 AI 도구가 읽어야 하는 규칙, 로직, 절차 | `docs/` |
| 특정 도구 전용 기능 (Claude Code 슬래시 커맨드 트리거 등) | 해당 도구 설정 디렉토리 |

**공통 내용을 도구 전용 디렉토리에 넣는 것은 금지.**
예: `.claude/`에 넣으면 Claude Code만 읽고 Codex/Cursor는 무시한다.

---

## 의견 요청 vs 실행 요청 구분

다음은 의견만 말하고 멈춘다. 코드/파일 수정 금지.

- "어떻게 생각해?"
- "어떤 게 나을까?"
- "이거 맞아?"
- "~하면 어떨까?"
- "뭐가 없을까?"

다음이 있어야 실행한다.

- "해줘", "적용해줘", "진행해줘", "수정해줘", "만들어줘"

---

## 핵심 원칙

코드를 작성하거나 수정하기 전에 반드시 계획과 근거를 먼저 설명하고,
사용자 승인을 받은 후에만 코드를 작성한다.

---

## 구현 요청일 때

1. **왜 이 설계인가** — 선택한 근거 (공식 문서, 라이브러리 설계 원칙 등 출처 포함)
2. **무엇을 해결하는가** — 어떤 문제를 풀고, 어떤 이점이 있는지
3. **어떻게 구현할 것인가** — 변경할 파일 목록과 각 파일에서 할 일
4. **대안과 비교** — 다른 방법 대비 이 방법을 선택한 이유 (선택적)

→ 설명 후 승인 대기 → 승인 시 코드 작성

---

## 버그 수정일 때

1. **왜 버그가 발생했는가** — 근본 원인 분석
2. **어떻게 수정하면 해결되는가** — 수정 근거와 작동 원리
3. **변경할 파일과 내용**

→ 설명 후 승인 대기 → 승인 시 코드 작성

---

## 금지

- 승인 없이 바로 코드 작성
- 근거 없이 "이렇게 하면 됩니다" 식의 설명
- 공식 문서나 출처 없이 설계 방향 단정

---

## 예외

다음과 같은 단순 작업은
승인 없이 바로 수정 가능하다.

- 오타 수정
- 문구 변경
- 스타일 값 수정
- 주석 수정
- 파일 이동 없이 수행 가능한 단순 리팩터링

---

## 설명 방식

- 초급 프론트엔드 개발자도 이해할 수 있게 설명한다.
- 어려운 용어 사용 시 의미를 함께 설명한다.
- 변경 이유를 우선 설명하고 구현 내용을 설명한다.
- 필요하다면 시각화도 같이 해준다.
- 참고문서 또는 블로그도 같이 첨부해준다.

---

## 변경 전 확인

코드를 수정하기 전에 다음 내용을 먼저 설명한다.

- 영향받는 파일
- 영향받는 기능
- 예상되는 부작용
- 테스트가 필요한 영역

변경 범위가 큰 경우에는 작업 계획을 단계별로 설명한다.

---

## 코드 작성 규칙

- 사용자가 명시적으로 "코드 줘", "구현해줘", "수정해줘"를 요청한 경우에만 실제 코드를 작성한다.
- 그 외에는 설계, 개념, 구현 방향을 우선 설명한다.

---

## 주석 규칙

코드를 작성할 때는 유지보수를 위해 적절한 주석을 함께 작성한다.

다만 코드 자체만으로 의미가 명확한 경우에는 불필요한 주석을 작성하지 않는다.

다음과 같은 경우 주석을 반드시 작성한다.

- 비즈니스 로직
- 복잡한 조건문
- API 호출 목적
- 상태 변경 이유
- 성능 최적화 로직
- 임시 우회 처리(workaround)
- 의도 파악이 어려운 구현

### 좋은 주석 예시

```ts
// 사용자가 로그인되어 있지 않으면 로그인 페이지로 이동
if (!user) {
  navigate('/login');
}
```

```ts
// 서버 부하를 줄이기 위해 검색어 입력 후 500ms 뒤에 요청
const debouncedKeyword = useDebounce(keyword, 500);
```

```ts
// 설문 응답 점수를 기반으로 사용자 성향 계산
const personalityType = calculatePersonality(answers);
```

### 나쁜 주석 예시

```ts
// count 변수 생성
const count = 0;
```

```ts
// 배열 반복
items.map(...)
```

코드가 무엇을 하는지가 아니라, 왜 이렇게 구현했는지를 설명하는 주석을 우선 작성한다.

## 문서 동기화

코드 변경 후 아래 기준으로 관련 docs를 즉시 업데이트한다.

| 코드 변경 | 업데이트할 문서 |
|-----------|----------------|
| 새 페이지 파일 추가 | `docs/pages.md`, `docs/progress.md` |
| 새 폴더 생성 (hooks/, services/ 등) | `docs/structure.md`, `docs/progress.md` |
| 새 패키지 설치 | `docs/stack.md` |
| 라우터 확정 | `docs/decision-log.md` (미결 → 결정 완료) |
| 기능 완성 | `docs/progress.md` (미구현 → 완료) |
| 기능 제거/변경 | 관련 docs 항목 삭제 또는 수정 |
| 환경변수 추가 | `docs/stack.md` 환경변수 목록 |

문서와 실제 코드가 다를 경우 사용자에게 알리고 업데이트를 제안한다.
전체 크로스체크가 필요하면 `src/`를 스캔한 뒤 위 표를 기준으로 확인한다.
